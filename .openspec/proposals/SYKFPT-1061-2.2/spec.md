# Spec: SPI & TDD Standards (SYKFPT-1061-2.2)

## 1. SPI 设计规范
- **单一职责**: 每个接口只负责一个具体的领域抽象（如路由、鉴权）。
- **无依赖逻辑**: 接口中严禁包含任何 `default` 实现逻辑。
- **线程安全**: 默认所有接口的实现类都应保证在 Java 21 虚拟线程环境下的并发安全性。

## 2. TDD 辅助规范
- 接口定义必须配合 `Test-Double` (Mock/Stub)。
- 每个方法必须配备详细的 Javadoc，描述前置条件、后置条件以及预期的异常行为，以便逻辑层开发者可以根据 Javadoc 编写 Mock 预期。

## 3. 异常规范
- 禁止直接透传底层中间件异常（如 JedisException）。
- 必须将其映射为 API 域的通用领域异常。
