# 核心受众、KPI 与负载预期 (Audience, KPIs & Scale)

## 核心受众
- **ISV 运维工程师**：负责分布式环境下的 Sidecar 部署、存储后端（MySQL/Redis）配置与监控。
- **ISV 开发工程师**：通过 Sidecar 提供的标准接口进行业务集成，无需关心 Token 刷新细节。
- **开放平台运营**：期望通过标准化边车降低生态接入的技术复杂度。

## 核心指标 (KPIs)
- **100% 分布式就绪**：支持通过配置切换至 MySQL/Redis/PG/SQLServer 存储，实现无状态容器化部署。
- **0 损耗升级**：确保 0.3.0 版本对 0.2.x 版本的本地文件配置支持 100% 兼容。
- **< 10ms 延迟增加**：本地缓存模式下，Sidecar 代理层带来的额外响应延迟不得超过 10ms。
- **99.9% 可用性**：通过分布式锁与数据库事务保证分布式环境下 Token 刷新的一致性与高可用。

## 负载预期 (Scale)
- **并发能力**：单集群支持最高 **500 QPS** 的 API 代理转发。
- **连接数**：支持同时维护超过 **10,000 个** 活跃账号的 Token 状态。
- **存储规模**：单库支持千万级消息/Token 记录存储。
