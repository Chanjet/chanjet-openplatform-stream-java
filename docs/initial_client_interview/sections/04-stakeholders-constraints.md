# 关键干系人与约束 (Stakeholders & Constraints)

## 关键干系人
- **ISV 开发者/运维**：系统的主要使用者，依赖边车进行 Token 管理和接口代理。
- **项目负责人**：决定 0.3.0 版本的演进方向与资源投入。

## 核心技术约束与要求
- **存储互斥性**：本地文件配置存储（Local）与共享存储（Shared）方式是**互斥**的，不可同时启用。
- **存储多样性**：必须支持 Redis, MySQL, PG, SQLServer。
- **混合模式**：支持将短效期数据（如缓存）放入 Redis，将长效期数据放入持久化存储。
- **向下兼容**：必须确保 0.3.0 版本对历史功能和旧版配置的平滑兼容。
