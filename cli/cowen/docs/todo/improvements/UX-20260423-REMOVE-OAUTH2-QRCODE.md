# 交互改进: 移除 OAuth2 初始化中的无效二维码 (UX-20260423-REMOVE-OAUTH2-QRCODE)

## 1. 背景与问题 (Context & Problem)
在 `cowen init` 的 OAuth2 模式下，系统会生成授权 URL 并同时渲染一个二维码。
- **无效性**: 扫码操作通常发生在手机等移动设备上。然而，OAuth2 授权完成后需要回调至本地监听端口 (`127.0.0.1:<PORT>`)。
- **交互冲突**: 移动设备无法访问发起命令的 PC 端的本地环回地址，导致回调失败、授权流程中断。
- **误导性**: 提供二维码暗示了支持“跨端授权”，但实际逻辑依赖于“同机浏览器环境”，这会误导用户并造成负面的初次使用体验。

## 2. 改进方案 (Proposed Improvements)
- **移除二维码渲染**: 在 `src/cmd/init.rs` 的 OAuth2 逻辑分支中，移除 `qrcode` 的生成与输出代码。
- **强化文字提示**: 明确告知用户必须在“当前机器”的浏览器中完成授权。
- **保留手动链接**: 继续保留打印出的完整 URL，以便在自动打开浏览器失效时，用户可以手动复制到本机浏览器。

## 3. 预期效果 (Expected Outcome)
- 消除无效的交互路径。
- 减少由于扫码回调失败导致的咨询和报错。
- 使初始化流程更加聚焦于本机自动化链路。

---
> [!TIP]
> 仅针对 OAuth2 授权 URL 移除二维码。如果未来有其他不需要本地回调的场景，可重新评估。
