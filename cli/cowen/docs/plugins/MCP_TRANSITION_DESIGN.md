# 基于底层 API 供给的渐进式 MCP 接入详细设计方案 (架构总览)

本文档旨在阐述如何通过 **“宿主统一接管代理 (Host Wrapper) + 宿主安全内存透传 (Memory Env Injection) + 渐进式空壳中继 (Generic Relay Plugin)”** 的全新演进架构，将 `cowen` 现有的底层流连接器能力极其纯粹、安全且零心智负担地暴露给 AI Agent（MCP Client）。

> **详细设计分支导读**：
> 针对不同的受众和物理执行层，本方案已将详细细节拆分为两篇独立的文档：

> *   **[插件设计方案 (PLUGIN_DESIGN.md)](file:///Users/zhangliang/chanjet/dev/workspace/open-streaming-connector/cli/cowen/docs/mcp/PLUGIN_DESIGN.md)**：面向三方多语言插件开发者。描述插件作为无状态空壳，如何声明 `plugin.json` 契约，并通过环境变量一键透传与宿主接口进行中继转发。

## 核心架构演进全景图与消息流向

为了保持 `cowen` 作为底座的高内聚与高安全性，系统剥离了传统重型插件的硬编码包袱，让三方插件退化为一个纯粹的 JSON-RPC 翻译通道，所有的业务逻辑与网络代发收敛于宿主内部。

### 核心收益 (Core Benefits)

1. **极致的开闭原则 (OCP)**：当需要为 `cowen` 增加新连接器工具时，**完全无需修改或发布任何三方插件代码**。宿主的动态注册表会在运行时渐进式地将新工具推送给 Agent。
2. **轻量生态与开发零摩擦**：插件开发者无需引入厚重的凭证管理、外网 API 依赖库或重型缓存逻辑，100-200 行代码即可完成无脑路由的桥接。
3. **物理级安全防串流**：底座常量（Endpoint，Token）彻底拒绝落盘，配合 `exclusive` 多 Profile 租户生命周期模型，从根源上杜绝了密钥泄露与租户间上下文串户。
