# Cowen CLI 更新日志 (Changelog)

所有针对 `cowen` 命令行工具的功能变更都将记录在本文档中。

---

## [0.1.5] - 2026-04-10

### 🚀 新特性 (Features)
- **动态 OpenAPI 发现 (Dynamic Spec Discovery)**: 引入服务驱动的规约同步机制。支持从静态端点下载完整 Spec，并具备基于权限列表的动态 Fallback 聚合能力。内置 1 小时 TTL 缓存，确保 API 列表与平台能力实时同步。
- **推送退避机制 (Push Backoff)**: 为 `appTicket/resend` 请求引入了指数退避算法。当检测到云端流量控制（HTTP 409）时，自动按阶梯延长重试等待时间，最高可达 24 小时。
- **隐私与功能控制 (Privacy & Controls)**: 新增 `--no-telemetry` 和 `--no-ai` 全局参数，支持显式关闭匿名遥测与本地 AI 语义搜索。
- **全局环境管理 (Global Profile Management)**: 为 `status` 和 `daemon` 命令新增 `--all` 参数，支持一键批量管理所有已配置环境的系统状态。
- **守护进程自动拉起 (Auto-Heal)**: 实现了“静默守卫”逻辑。在执行业务命令时，若检测到守护进程缺失，工具会自动在后台静默恢复运行。

### 🔧 改进与修复 (Improvements & Fixes)
- **构建体系标准化 (Makefile Refinement)**: 
  - 统一构建目标命名为专业标准（`aarch64`, `x86_64`）。
  - 构建产物现在包含 **SHA1** 和 MD5 校验和。
  - 优化了产物目录结构，区分 build (短文件名) 与 package (带版本长文件名)。
- **SDK 鲁棒性增强**:
  - **URL 编码修复**: 修复了 WebSocket 握手 URL 手动拼接导致的特殊字符（如 `#`, `&`）截断问题，确保身份参数完整传递。
  - **凭据空检查 (Fail-fast)**: 在 SDK 启动与 CLI 维护循环中增加凭据非空校验，彻底杜绝发送 `app_key=""` 的无效请求。
  - **依赖升级**: 全面升级 `tokio` (1.51), `tokio-tungstenite` (0.29), `reqwest` (0.13) 等核心库，提升异步性能与安全性。
- **AI 语义搜索优化**: 修正了索引新鲜度检查后缀逻辑（`.json` -> `.yaml`），大幅降低了重复重建索引带来的 CPU 开销。
- **日志与安全脱敏**:
  - 实现了全链路日志掩码，自动对日志中的敏感参数进行脱敏处理。
  - 将守护进程状态消息重定向至 `stderr`，确保 `stdout` 仅输出业务数据。
- **环境隔离加固**: 增强了 `log list` 的隔离性，并强化了 `reset` 指令，支持物理清除专有 DLQ 目录及历史滚动日志。

## [0.1.4] - 2026-04-09

### 🚀 新特性 (Features)
- **版本溯源增强**: 命令行版本号现在包含源码 Git Hash 值。
- **Linux 语义搜索支持**: 发布了针对 `linux/x86_64` 架构的“满血”版本，内置 AI 推理引擎。

### 🔧 改进与修复 (Improvements & Fixes)
- **Windows API 获取修复**: 解决了 Windows 环境下分页偏移导致的 API 列表拉取缺失问题。
- **运行稳定性增强**: 提升了后台守护进程的可靠性，增强了断线后的秒级自愈能力。

---

## [0.1.3] - 2026-04-07

### 🚀 新特性 (Features)
- **Windows 原生安装器**: 引入了原生的 `setup.exe` 安装程序。
- **API 列表展示增强**: 刷新规约时直观显示接口总数对比。

### 🔧 改进与修复 (Improvements & Fixes)
- **分页交互优化**: 支持更直观的 `-n` (page-size) 分页查询。
- **守护进程重启修复**: 解决了由于残留进程导致的重启死循环问题。
