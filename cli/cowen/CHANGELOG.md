# Cowen CLI 更新日志 (Changelog)

所有针对 `cowen` 命令行工具的功能变更都将记录在本文档中。

---

## [0.1.5] - 2026-04-10

### 🚀 新特性 (Features)
- **隐私与功能控制 (Privacy & Controls)**: 新增 `--no-telemetry` 和 `--no-ai` 全局参数，允许用户显式关闭匿名的运行状态上报与本地的 AI 语义搜索功能。在工具初次运行时，也会明确展示《安全与隐私提示》。
- **全局环境管理 (Global Profile Management)**: 为 `status` 和 `daemon start/stop/restart` 命令新增 `--all` 参数，支持一键扫描并批量管理所有已配置环境的系统状态与守护进程，大幅提升多环境场景下的运维效率。
- **守护进程自动拉起 (Auto-Heal)**: 实现了“静默守卫”逻辑。如果电脑重启或守护进程因意外停止，用户在执行任意业务命令（如 `api list`, `status`）时，工具会自动在后台重新拉起守护进程，无需手动干预。

### 🔧 改进与修复 (Improvements & Fixes)
- **诊断系统增强**: 修复了 `status --all` 在遇到损坏环境时静默跳过的 Bug，现在会准确汇总错误环境；同时优化了 `profile list`，自动过滤 OpenAPI 规约缓存等非环境文件。
- **安全日志脱敏 (Security Hardening)**: 实现了全链路日志掩码。无论是 API 调用路径中的敏感 Query 参数，还是请求体（Body）中的私密数据，均会在 `sys.log` 和 `audit.log` 中被自动脱敏，杜绝了敏感信息在本地日志中的意外留痕。
- **环境隔离与彻底清理**: 
  - 增强了 `log list` 的隔离性，用户现在只能在所属 Profile 下查看相关的全局日志和专有日志。
  - 强化了 `reset` 指令，现在会同步物理清除专有的死信队列（DLQ）目录及所有历史滚动日志。
- **版本透明化**: 修复了版本显示逻辑，现在 `--version` 能准确反映 Git Hash。
- **测试体系建设与集成发版保障**: 完成了单元测试与功能命令行行为的黑盒集成测试基座。并在自动发布流水线末端挂载了自动的“健康验证”保险丝，并新增了针对安全性与隔离性的探索性测试用例。

## [0.1.4] - 2026-04-09

### 🚀 新特性 (Features)
- **版本溯源增强**: 命令行版本号现在包含源码 Git Hash 值，便于精准追踪构建版本。
- **Linux 语义搜索支持**: 发布了针对 `linux/amd64` 架构的“满血”版本，内置 AI 推理引擎，支持在 Linux 环境下通过自然语言进行 API 语义搜索。

### 🔧 改进与修复 (Improvements & Fixes)
- **Windows API 获取修复**: 解决了在 Windows 环境下由于分页偏移导致的 API 列表拉取缺失问题，确保了多平台数据的一致性。
- **状态报告准确性修复**: 修复了 `status` 命令中 `AppTicket` 接收时间显示不准的问题，现在能真实反映安全凭据的更新历史。
- **运行稳定性增强**: 
  - 显著提升了后台守护进程的可靠性，解决了在极端网络环境下连接“假死”导致的指令超时问题。
  - 增强了断线后的秒级自愈能力，确保流桥连接在服务端重启后能迅速自动恢复。

---

## [0.1.3] - 2026-04-07

### 🚀 新特性 (Features)
- **Windows 原生安装器**: 引入了原生的 `setup.exe` 安装程序，提升了 Windows 环境下的部署体验。
- **API 列表展示增强**: 使用 `api list -r` 强制刷新时，现在会直观地显示刷新前和刷新后的接口总数对比。

### 🔧 改进与修复 (Improvements & Fixes)
- **分页交互优化**: 精简了 `api list` 参数，支持更直观的 `-n` (page-size) 分页查询。
- **守护进程重启修复**: 彻底解决了由于残留进程导致的重启死循环问题。
