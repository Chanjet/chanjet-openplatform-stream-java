# Cowen CLI 更新日志 (Changelog)

所有针对 `cowen` 命令行工具的功能变更都将记录在本文档中。

---

## [0.2.0] - 2026-04-21

### 🚀 新特性 (Features)
- **全自动身份验证支持**: 引入了更便捷的 OAuth2 授权机制，通过自动化流程处理令牌获取，显著降低了初次配置的复杂度。
- **守护进程自启动支持**: 新增 `cowen daemon service` 命令，支持在 macOS、Linux 及 Windows 上一键配置守护进程的开机自启动。
- **安装过程自动激活服务**: 优化了全平台安装体验，在安装 Cowen CLI 时将自动完成自启动服务的配置与激活，实现即装即用。
- **Profile 智能管理体系**:
    - **环境一键迁移**: 新增 `cowen profile rename` 命令，支持配置环境及其凭据的完整迁移。
    - **自动初始化优化**: 支持自动化的环境命名机制，并强化了保护措施，防止误覆盖现有配置。
- **轻量化守护进程**: 重构了后台服务逻辑，通过组件拆分提升了运行效率，支持更灵活的按需启动模式。
- **全平台补全增强**: 扩展了对 PowerShell 的原生支持，并优化了命令行补全的自动安装体验。

### 🔧 改进与修复 (Improvements & Fixes)
- **初始化稳定性加固**: 优化了配置保存机制，彻底解决了特定场景下配置文件可能丢失的问题。
- **状态诊断增强**: 升级了 `cowen status` 报告系统，提供更直观、更清晰的多级状态反馈。
- **配置发现机制优化**: 优化了扫描算法，确保在各种环境下都能准确识别并加载已有的配置。
- **诊断系统鲁棒性升级**: `status` 指令集成了多级错误感知能力，能精确捕获采集异常并提供详尽的修复指导。
- **API 交互能力修复**: 解决了在特定模式下获取接口列表与规约时的权限校验拦截问题。
- **发布质量保障**: 引入了更严格的发布验证机制，确保全平台交付产物的可靠性与功能完整性。

## [0.1.5] - 2026-04-11

### 🚀 新特性 (Features)
- **动态服务规约同步**: 引入了自动化的接口定义同步机制，确保本地 API 列表与平台能力实时对齐，支持在复杂的网络环境下自动降级与恢复。
- **推送频率退避控制**: 实现了智能的重试退避算法，在遭遇云端流量控制时能自动调整压力，保障系统的长期运行稳定性。
- **隐私与功能开关**: 新增全局参数，支持用户根据需求显式关闭匿名遥测与本地语义搜索功能。
- **全局环境批量管理**: 支持通过单一指令查看或管理所有已配置环境的运行状态。

### 🔧 改进与修复 (Improvements & Fixes)
- **后台服务自愈能力加固**: 显著提升了守护进程的稳定性，确保在系统意外崩溃或异常重启后，能以前所未有的可靠性重新上线。
- **凭证获取体验优化**: 解决了在特定竞态条件下出现的凭据获取延迟问题，大幅缩短了从初始化到服务就绪的等待时间。
- **配置校验预检机制**: 增加了启动时的强制逻辑校验，能即时识别凭据配置错误并提供修正建议，避免无效的后台运行。
- **安装包命名标准化**: 统一并规范了全平台的构建产物命名，集成了更严密的产物完整性校验流程。
- **日志与安全脱敏**: 进一步强化了全链路日志的敏感数据自动掩码功能，确保调试信息不泄露。
- **环境隔离加固**: 强化了多实例并存时的资源清理与数据隔离逻辑。

## ⚠️ 已知问题 (Known Issues)

- **Windows 更新后进程挂死**: 在 Windows 系统更新或重启后，旧的 `cowen` 进程可能无法正常退出，导致新进程无法启动。目前需要手动在任务管理器中结束残留的 `cowen` 进程。详细记录参考 [BUG-20260423-WINDOWS-UPDATE-STUCK](docs/bugs/BUG-20260423-WINDOWS-UPDATE-STUCK.md)。
