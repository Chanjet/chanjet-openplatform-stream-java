# Cowen CLI 更新日志 (Changelog)

所有针对 `cowen` 命令行工具的重要变更都将记录在本文档中。

---

## [0.1.4] - 2026-04-08

### 🚀 新特性 (Features)
- **Linux amd64 满血版分发**: 提供了针对 `linux/amd64` 架构的优化构建版本，包含完整的 ONNX Runtime AI 推理引擎。

### 🔧 改进与修复 (Improvements & Fixes)
- **版本同步与稳定性增强**: 
  - 同步升级至 0.1.4 以匹配最新的 Streaming Connector 系统架构。
  - **核心修复**: 协同解决了服务端路由 TTL 刷新失效以及 SDK 在半连接状态下的自愈能力，显著提升了 CLI 在生产环境中的长连稳定性。

---

## [0.1.3] - 2026-04-07

### 🚀 新特性 (Features)
- **Windows 原生安装器 (Installer)**: 废弃了旧版的 `install.ps1` 脚本，全面引入原生的 `cowen_setup` 安装程序以进行 Windows (`win64`) 环境的打包与自动安装分发。
- **API 列表展示增强**: 使用 `api list -r` 强制刷新时，现在会直观地显示刷新前和刷新后的接口总数对比。
- **异步遥测系统 (Telemetry)**: 
  - 实现基于 `tokio::spawn` 的非阻塞上报引擎。
  - 支持 `cli_first_run` (激活) 和 `command_run` (活跃) 事件上报。
- **设备指纹识别**: 集成匿名硬件指纹生成，支持精确的 DAU 统计。

### 🔧 改进与修复 (Improvements & Fixes)
- **API 列表分页逻辑精简**: 重构了 `api list` 命令的参数，移除了冗余的 `top` 限制参数，并将 `-n` 直接映射为 `page-size`，使得命令行的分页查询交互更加直观（默认 `page=1`, `page-size=20`）。
- **云端规范获取分页修复**: 修正了在后台向开放平台拉取全量接口列表时的接口调用参数，从 `currentPage` 切换为基于 0 索引的 `page` 参数，确保能完整无漏地拉取所有云端接口。
- **守护进程 (Daemon) 重启逻辑修复**: 彻底解决了由于僵尸进程导致 `api list -r` 触发的无限重启死循环问题。现在可以精确识别并安全清理所有关联的残留进程。
- **接口发现 (Discovery) 逻辑修复**: 解决了在动态接口发现过程中丢失部分 API 规范的问题。引入深度合并逻辑（确保同一路径下的多个 HTTP Method 被正确保留），并兼容解析基于 JSON 字符串返回的 `openApi` 规范字段。
- **接口强制刷新优化**: 重构了 `get_openapi_spec` 获取逻辑，确保当使用强制刷新标志 (`-r`) 时能被严格遵守，彻底绕过本地文件缓存，确保获得云端的最新规约。
- **User-Agent 规范化**: 建立统一的 HttpClient 工厂，所有请求现在携带标准 UA: `Cowen/v0.1.3 (OS; ARCH)`。
- **优雅退出**: 增加 50ms 退出延迟，确保异步上报请求有足够时间发出。
- **静默失败**: 优化网络异常处理，所有遥测报错在用户端均保持静默。
